## 2. Mechanical Invariants

1. **Theme is auto-generated by Nix, never hand-edited.**
   `modules/widgets/quickshell.nix` serializes `config.color`, `config.font`,
   `config.ui`, and `config.home` into `globals/Theme.qml` as a QML
   `Singleton`. Any QML file imports it via `import globals` and references
   `Theme.color.accent`, `Theme.spacing.s4`, `Theme.font.sans`, etc.
   **Never inline design tokens in QML.**
2. **QML imports use qmldir registries.** Components, widgets, services, and
   windows are each registered in their own `qmldir` file. Import by type
   name, not file path. The `QML2_IMPORT_PATH` env var in the systemd
   service points to the quickshell directory.
3. **One widget per file.** Each `widgets/*.qml` is a self-contained
   element. Compose them in `windows/TopBar.qml` or `windows/RightModule.qml`.
4. **Reuse components — don't reinvent.** The component library (`Card`,
   `Chip`, `Icon`, `Label`, `Trigger`) covers the standard UI primitives.
   Before creating a new visual element, check if composing existing
   components suffices.
5. **Services poll, widgets render.** Background services (`services/`) own
   the polling logic and expose properties. Widgets bind to those properties
   and render. Never put polling logic inside a widget.
6. **X-Switch-Trigger for cache busting.** The systemd service in
   `quickshell.nix` sets `X-Switch-Trigger = builtins.hashString "sha256"
   themeQmlText` — any design token change triggers a service restart.

## 3. Component Catalog

| Component | File | Purpose |
|---|---|---|
| `Card` | `Card.qml` | Surface wrapper with `variant` (surfaceVariant/accent), hover scale, border color animation, `clicked` signal. Uses `WrapperRectangle` from Quickshell. |
| `Chip` | `Chip.qml` | Compact label container for tags/filters |
| `Icon` | `Icon.qml` | Renders icons from the icon pack |
| `Label` | `Label.qml` | Styled text using Theme.font.* tokens |
| `Trigger` | `Trigger.qml` | Wraps any element with click/hover interaction |

### Component Design Conventions

- **All visual properties reference `Theme.*`** — colors, spacing, radii,
  fonts, animation durations. Zero hardcoded values.
- **Animation timing:** `Theme.animDuration.f` (200ms) for fast transitions,
  `Theme.animDuration.n` (300ms) for normal. Use `Easing.OutQuad`.
- **Hover pattern:** increase opacity by 0.25, scale by 3%, border opacity
  by 0.3. Apply via `Behavior on <property>` with `ColorAnimation` or
  `NumberAnimation`.
- **Color construction:**
  ```qml
  var c = Qt.color(baseColor);
  return Qt.rgba(c.r, c.g, c.b, alpha);
  ```
  Alpha values flow from `Theme.effects.surfaceAlpha`,
  `Theme.effects.surfaceVariantAlpha`, or `Theme.effects.borderOpacity`.

## 4. Service → Widget Data Flow

```
Service.qml (polls system)     Widget.qml (renders UI)
─────────────────────────      ─────────────────────
Timer → property update   →    Binding to service.property
e.g. VolumeService.volume →    Volume.qml displays slider
```

Services are singletons instantiated at app startup. Each exposes typed
properties (int, string, bool) that widgets bind to. Services live in
`services/`, are registered in `services/qmldir`.

## 5. Adding a New Widget

1. **If it needs system data:** create a `services/NewService.qml` with a
   `Timer` polling the relevant system state. Register in
   `services/qmldir`.
2. **Create `widgets/NewWidget.qml`** — import `globals` for `Theme`,
   import components as needed, bind to service properties.
3. **Register in `widgets/qmldir`** with type declaration.
4. **Instantiate** in `windows/TopBar.qml` or `windows/RightModule.qml`.

## 6. Systemd Service

Defined in `modules/widgets/quickshell.nix`:

| Field | Value |
|---|---|
| Type | `simple` |
| Restart | `always`, 3s cooldown |
| Environment | `QML_DISABLE_DISK_CACHE=1`, `QML2_IMPORT_PATH` |
| ExecStart | `qs -p <quickshell-dir>` via bash |
| WantedBy | `graphical-session.target` |
| PartOf | `graphical-session.target` |

The `X-Switch-Trigger` hash ensures any config change triggers a restart.
`QML_DISABLE_DISK_CACHE=1` prevents stale QML bytecode during development.

## 7. Common Pitfalls

- **Theme.qml not found at runtime.** Ensure `QML2_IMPORT_PATH` includes the
  quickshell directory. Rebuild with `home-manager switch` to regenerate
  `globals/Theme.qml`.
- **QML cache poisoning.** Always keep `QML_DISABLE_DISK_CACHE=1` in dev.
  Stale `.qmlc` files cause invisible errors — changes don't take effect.
- **Animation jank.** Keep transitions under 300ms. Use
  `Theme.animDuration.f` (200ms) for most bar interactions.
- **Design token drift.** A hardcoded color or spacing in a QML file is a
  bug. Search for `"#` and `px` in QML files to catch drift.
- **Service property type mismatch.** If a widget binding fails silently,
  check that the service property type (int/string/bool) matches what the
  widget expects.
